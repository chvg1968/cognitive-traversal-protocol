# Contributing

Thanks for considering a contribution to the Cognitive Traversal Protocol.

This project is intentionally small. Contributions should make the protocol clearer, more useful, or easier to adopt without making it heavy.

## Good Contributions

- Clearer wording
- Better examples
- Translations
- Tool-specific usage notes
- Real-world case studies
- Accessibility improvements to the visual guide
- Corrections to templates or contradictions

## Contribution Principles

Before opening a pull request:

1. Keep the protocol universal.
2. Do not tie examples to one vendor, stack, or industry unless clearly labeled.
3. Preserve the rule that unknown impact blocks intervention.
4. Keep templates copy-paste friendly.
5. Prefer concrete examples over abstract claims.
6. Avoid adding dependencies unless there is a strong reason.

## Suggested Pull Request Shape

```md
## Summary
- What changed?

## Why
- What problem does this solve?

## Impact
- Directly affected files
- Indirectly affected readers or workflows

## Verification
- What did you check?

## Open Questions
- Anything still unresolved?
```

## Local Review

For documentation changes:

- Check links.
- Search for overly specific language.
- Confirm templates still match across README and docs.

For `index.html` changes:

- Open the file in a browser.
- Check desktop and mobile widths.
- Confirm internal navigation still works.
